Glosario de HTML

Cómo es el funcionamiento del internet y de los sitios web

  • Cliente:

    Una computadora o dispositivo que intenta acceder a internet.

  • Dirección IP:

    Identificador único asignado a cada dispositivo en una red para facilitar la comunicación.

  • fibras ópticas:

    Medio de transmisión de datos que utiliza pulsos de luz a través de hilos delgados de vidrio o plástico.

  • HTTP (Protocolo de Transferencia de Hipertexto):

    Protocolo estándar utilizado para la transferencia de datos en la World Wide Web.

  • Internet:

    Gran red de dispositivos interconectados que permite la transferencia de datos a nivel mundial.

  • Navegador:

    Aplicación de software que permite a los usuarios acceder y visualizar información en la web.

  • Página Web:

    Documento o recurso disponible en la World Wide Web accesible a través de un navegador.

  • Servidor:

    Computadora que almacena y sirve archivos y datos a los clientes que acceden a través de internet.

  • World Wide Web:

    Sistema de información en el que documentos y recursos están disponibles a través de internet.

Diferencias entre Front end y Back end

  • API (Interfaz de Programación de Aplicaciones):

    Conjunto de reglas que permite que dos software se comuniquen entre sí.

  • Backend:

    Parte del desarrollo web que se ocupa del lado del servidor, almacenamiento de datos y asegurar el funcionamiento del lado del cliente

  • CSS (Hojas de Estilo en Cascada):

    Lenguaje utilizado para establecer la apariencia y estilos de un documento HTML.

  • Desarrollo Web:

    Proceso de construcción y mantenimiento de sitios web o aplicaciones web.

  • Lenguaje de Programación:

    Conjunto de instrucciones que una computadora puede entender y ejecutar.

  • Framework (en el contexto del desarrollo web):

    Estructura conceptual y conjunto de herramientas que facilita el desarrollo de software.

  • Responsive:

    Diseño web que se adapta y presenta correctamente en diferentes tamaños de pantalla y dispositivos.

  • Selectores (en el contexto de CSS):

    Patrones que seleccionan los elementos HTML a los que se les aplicarán estilos.

  • URL (Localizador de Recursos Uniforme):

    Dirección que identifica un recurso en internet, como una página web.

  • Usuario:

    Persona que interactúa con el frontend de una aplicación o sitio web.

  • Web App (Aplicación Web):

    Aplicación que se accede a través de un navegador web.

Cómo se visualizan HTML, CSS y Javascript

  • CodePen:

    Plataforma en línea que permite la creación y edición de código HTML, CSS y JavaScript en un entorno de desarrollo integrado.

  • Código:

    Conjunto de instrucciones escritas en un lenguaje de programación.

  • Evento (en el contexto de JavaScript):

    Acción detectada por el navegador, como un clic o una pulsación de tecla, que puede activar una función en JavaScript.

  • Ícono:

    Pequeña representación gráfica que simboliza una función o acción en una interfaz de usuario.

  • Imágen:

    Elemento visual que se muestra en una página web, puede ser estática o dinámica.

  • Inspector (en el contexto de desarrollo web):

    Herramienta que permite inspeccionar y analizar la estructura y estilos de una página web.

  • Interactividad:

    Capacidad de una página web para responder a las acciones del usuario, como clics y movimientos.

  • Margen (en el contexto de CSS):

    Espacio entre el borde de un elemento y su contenido, que puede modificarse para ajustar la disposición en la página.

Primeras etiquetas de un proyecto

  • < a>, < /a > (hipervínculo):

    Etiqueta utilizada para crear hipervínculos a páginas web o secciones de la misma página. Se le asigna el atributo "href" con la dirección de destino.

  • alt (texto alternativo):

    Atributo de la etiqueta img que proporciona un texto descriptivo para la imagen. Se utiliza para accesibilidad y en casos de errores de carga de la imagen.

  • < body>, < /body> (cuerpo):

    Etiqueta que contiene todo el contenido visible de una página HTML. Define el cuerpo del documento.

  • < div>< /div> (división):

    Etiqueta utilizada como contenedor genérico para agrupar y estructurar elementos en secciones en una página web.

  • h1, h2, h3, h4, h5, h6 (encabezados):

    Etiquetas que definen encabezados o títulos, con h1 siendo el más importante y h6 el menos importante. Proporcionan una estructura jerárquica al contenido.

  • (línea horizontal):

    Etiqueta que crea una línea horizontal para separar o dividir secciones en una página HTML.

    Etiqueta que envuelve todo el contenido HTML de un documento. Indica el comienzo y el final del código HTML.

  • < img/> (imagen):

    Etiqueta utilizada para insertar imágenes en una página HTML. Se le asigna el atributo "src" con la dirección de la imagen.

  • < li>< /li> (elemento de lista):

    Etiqueta que define un elemento de lista dentro de una lista ordenada (ol) o desordenada (ul).

  • < ol>< /ol> (lista ordenada):

  • < p>< /p> (párrafo):

    Etiqueta que define un párrafo de texto en una página HTML.

  • < span>< /span>:

    Etiqueta utilizada para aplicar estilos o marcar secciones de texto en línea dentro de un bloque de texto.

  • src (fuente):

    Atributo de la etiqueta img que especifica la fuente o dirección de la imagen.

    • (lista desordenada):

    Elementos de bloque y línea

    • Elementos de bloque:

      Los elementos de bloque en HTML son aquellos que ocupan todo el ancho disponible en su contenedor principal y siempre inician en una nueva línea. Estos elementos estructuran y organizan el contenido de la página, creando bloques visuales distintos. Los elementos de bloque pueden contener otros elementos, tanto de bloque como en línea. Ejemplos de elementos de bloque incluyen div , p , h1 a h6 , ul , ol , li , table , form , entre otros. Estos elementos son fundamentales para establecer la estructura y el diseño de la página web.

    • Elementos de Línea:

      Los elementos de línea, por otro lado, son aquellos que ocupan solo el espacio necesario en la línea y no inician en una nueva línea. Estos elementos se alinean horizontalmente y permiten que otros elementos de línea compartan la misma línea en el diseño de la página. Ejemplos de elementos de línea incluyen span>, a>, strong>, em>, img>, input>, br>, entre otros. Los elementos de línea son útiles para aplicar estilos o contenido en línea, sin afectar la estructura general de la página. La combinación adecuada de elementos de bloque y de línea es esencial para lograr un diseño web coherente y efectivo.

    Elementos variados de HTML

    • < b >< /b>:

      Etiqueta utilizada para resaltar o hacer que el texto sea más fuerte visualmente.

    • < br >:

      Etiqueta de línea que provoca un salto de línea en el texto. Se utiliza para forzar un cambio de línea sin iniciar un nuevo párrafo.

    • < em >< /em>:

      Etiqueta utilizada para resaltar o enfatizar el texto. Generalmente se muestra en cursiva.

    • < hr >:

      Etiqueta de bloque que crea una línea horizontal en la página, utilizada para separar secciones. No tiene etiqueta de cierre y se escribe como < hr>.

    • < small >< /small>:

      Etiqueta de línea utilizada para representar texto más pequeño en tamaño.

    • < strong >< /strong>:

      Etiqueta de línea que indica que el texto marcado debe ser considerado importante. Se utiliza con la etiqueta < strong> y generalmente se muestra en negrita.

    • Sub (Subscript) y Sup (Superscript):

      Etiquetas de línea que representan texto como subíndice (< sub>) o superíndice (< sup>), respectivamente.

    • comentarios:

      Comentarios en el código HTML que no son visibles en la página. Se escriben con < !-- comentario -->. O con el shortcut “cmd+shift+7” o “ctrl+shift+7”

    Entidades de HTML

    • Entidades HTML:

      Las entidades HTML son códigos especiales que se utilizan para representar caracteres reservados o caracteres especiales en un documento HTML. Estos códigos permiten mostrar correctamente ciertos caracteres que de otra manera podrían causar problemas de interpretación en el navegador o podrían confundirse con elementos del lenguaje HTML. Se utilizan mediante el uso de la apertura de ampersand y el cierre con punto y coma.

    ¿Qué es CSS 3?

    • Atributo:
      • Elemento utilizado en HTML para proporcionar información adicional sobre un elemento y, en este contexto, se menciona en relación con la aplicación de estilos en CSS.
    • CSS (Cascading Style Sheets):
      • Lenguaje de estilos utilizado para describir el aspecto y formato de un documento escrito en HTML.
    • CSS externo:
      • Método de incluir estilos CSS en un documento HTML mediante la vinculación de una hoja de estilos externa a través de la etiqueta link.
    • CSS interno
      • Método de incluir estilos CSS directamente dentro del documento HTML utilizando la etiqueta style en la sección head.
    • CSS en línea
      • Método de incluir estilos CSS directamente dentro de un elemento HTML utilizando el atributo style.
    • Elemento HTML:
      • Componente básico de una página web, sobre el cual se aplican reglas CSS para definir su estilo.
    • Propiedad:
      • Característica específica de estilo que se puede modificar mediante CSS, como color, tamaño, fuente o posición.
    • Selector:
      • Patrón que define a qué elementos HTML se aplicarán las reglas CSS.
    • Sintaxis:
      • La estructura y el formato correctos que debe seguir un conjunto de reglas CSS, compuesto por selectores, propiedades y valores.
    • Specificity:(Especificidad)
      • Término que indica cómo se resuelve un conflicto cuando dos o más reglas CSS se aplican al mismo elemento, basado en la especificidad de las reglas.
    • Valor:
      • Información específica que se asigna a una propiedad en CSS para definir el estilo de un elemento.

    Propiedades de color y sus sistemas

    • background-color:
      • Propiedad de CSS que define el color de fondo de un elemento en una página web.
    • Color
      • Propiedad de CSS que establece el color del texto de un elemento en una página web.
    • Hexadecimal
      • Sistema de representación de colores que utiliza una combinación de números y letras (0-9, A-F) para indicar los valores de rojo, verde y azul.
    • RGB (Red, Green, Blue):
      • Modelo de color que define un color indicando la cantidad de rojo, verde y azul que se mezclan para obtener dicho color.
    • Sistema de color:
      • Conjunto de reglas y valores utilizados para representar y definir colores en diseño web.
    • Word keywords:
      • Palabras clave en inglés que representan colores específicos en CSS, como black, white o red.

    Propiedades de texto

    • font-family:
      • Propiedad de CSS que permite asignar una tipografía al texto.
    • Google fonts:
      • Servicio gratuito de Google que proporciona una amplia variedad de tipografías para su uso en páginas web.
    • letter spacing:
      • Propiedad de CSS que establece el espaciado entre letras en un texto.
    • line heights:
      • Propiedad de CSS que determina el espacio vertical entre líneas de texto.
    • Serif:
      • Tipo de tipografía con remates o adornos en las letras.
    • text-align:
      • Propiedad de CSS que define la alineación horizontal del texto en un elemento. Puedes usar valores como left, right, center o justify.
    • text-decoration:
      • Propiedad de CSS que agrega decoración al texto, como subrayado o tachado.
    • text-transform:
      • Propiedad de CSS que controla la capitalización del texto, convirtiéndolo en mayúsculas, minúsculas o capitalizando cada palabra.
    • vertcial-align:
      • Propiedad de CSS que controla la alineación vertical de un elemento.
    • word-spacing:
      • Propiedad de CSS que establece el espaciado entre palabras en un texto.
    • word-wrap:
      • Propiedad de CSS que controla si el texto debe envolverse en múltiples líneas.
    • x-height:
      • La altura de las letras minúsculas sin considerar ascenderes o descenderes.

    Unidades de CSS

    • Absolute units:
      • Unidades absolutas en CSS, como píxeles, puntos, centímetros, pulgadas y milímetros, que no dependen de un valor de referencia y son fijas.
    • em:
      • Unidad relativa en CSS que se calcula en función del tamaño de la fuente del elemento padre.
    • Percentages: (%)
      • Unidades relativas en CSS que se calculan en función de otro valor, generalmente el tamaño del elemento padre.
    • Pixel: (px)
      • Unidad absoluta en CSS utilizada para medir el tamaño de elementos en una página web, como texto, imágenes y márgenes.
    • rem:
      • Unidad relativa en CSS que se calcula en función del tamaño de la fuente del elemento raíz (generalmente el elemento <html>).
    • vh (Viewport Height):
      • Unidad relativa en CSS que representa el porcentaje del tamaño de la ventana gráfica en términos de altura.
    • vw (Viewport Width):
      • Unidad relativa en CSS que representa el porcentaje del tamaño de la ventana gráfica en términos de ancho.

    Box model

    • Border:
      • Línea que rodea la caja y se utiliza para la apariencia. Puede mostrar líneas con colores y tipos diferentes.
    • Box model:
      • Concepto que establece que todos los elementos en los navegadores se construyen a partir de cajas. Cada elemento HTML se muestra como una caja rectangular con áreas internas y externas.
    • Content:
      • Área interna del Box Model que contiene el contenido real del elemento, como texto e imágenes. Su tamaño se puede modificar con la propiedad width.
    • Display:
      • Propiedad CSS que determina el tipo de elemento y cómo se presenta. Puede ser block, inline, o valores específicos como table.
    • Margin:

    👉 Conoce qué es Git, GitHub y Control de Versiones
    Branch:
    Una rama en Git es una línea de desarrollo independiente que permite a los equipos trabajar en paralelo en diferentes características o correcciones de errores.
    Commit:
    Un commit en Git representa un conjunto de cambios realizados en los archivos del proyecto. Cada commit tiene un mensaje que describe los cambios realizados.
    Control de versiones:
    Práctica que implica gestionar y rastrear los cambios realizados en el código fuente a lo largo del tiempo. Permite mantener un historial de modificaciones, facilitando la identificación de errores, la colaboración y la reversión a versiones anteriores.
    Fork:
    Hacer una copia independiente de un repositorio en GitHub bajo tu propia cuenta. Los cambios en un fork no afectan al repositorio original hasta que se envían solicitudes de extracción.
    Git:
    Sistema de control de versiones distribuido ampliamente utilizado. Permite realizar un seguimiento de los cambios en el código fuente, colaborar con otros desarrolladores y gestionar proyectos de software de manera eficiente.
    GitHub:
    Plataforma basada en la nube que utiliza Git para alojar repositorios de código fuente. Facilita la colaboración, seguimiento de problemas, implementación y otras funciones relacionadas con el desarrollo de software.
    Merge:
    Fusionar cambios de una rama a otra en Git. Los merges combinan líneas de desarrollo separadas en una sola línea de desarrollo.
    Pull Request:
    Una solicitud que un desarrollador envía a un repositorio para solicitar la incorporación de sus cambios. Es común en GitHub y proporciona una forma de revisar y discutir los cambios antes de fusionarlos.
    Push:
    Enviar cambios locales a un repositorio remoto en Git. Los cambios realizados en una máquina local se reflejan en el repositorio compartido.
    Registro de cambios:
    Historial que documenta las modificaciones realizadas en el código fuente a lo largo del tiempo. Facilita la identificación de contribuciones, la solución de problemas y el seguimiento de la evolución del proyecto.
    Repositorio:
    Almacén centralizado que contiene archivos y carpetas gestionados por un sistema de control de versiones. Puede ser local o remoto, como en el caso de GitHub, donde múltiples desarrolladores colaboran en un proyecto.
    Software de control de versiones:
    Herramienta que ayuda a los equipos de desarrollo a gestionar y rastrear los cambios en el código fuente. Facilita la colaboración, la identificación de errores y el mantenimiento de un historial de versiones del proyecto. Git es un ejemplo de software de control de versiones.

    Javascript I

    Conoce qué es JavaScript
    ECMAScript
    Es un estándar del lenguaje de JavaScript. Se desarrolla según las indicaciones de un documento proporcionado por Ecma International. Define las características y la implementación del lenguaje JavaScript.
    Vanilla JavaScript
    Es un término utilizado para referirse a JavaScript puro, sin ninguna librería adicional como jQuery o frameworks como Angular. Dominar los fundamentos de JavaScript sin dependencias externas.
    JavaScript
    Es un lenguaje de programación que permite implementar funciones complejas en una página web. Añade funcionalidad interactiva y contenido dinámico a los sitios web.
    Identifica cuáles son los Data Types
    Booleanos
    Valores que consisten en true o false, representando condiciones lógicas.
    NaN (Not a Number)
    Representa un valor que no es un número. Puede generarse en operaciones que no resultan en un número válido.
    Cadenas (Strings)
    Secuencias de caracteres entre comillas, como texto. Pueden representarse con comillas simples o dobles.
    Null
    Representa intencionalmente un valor nulo o vacío. Indica la ausencia de valor.
    Number
    Tipo de dato que admite números positivos, negativos y decimales.
    Undefined
    Representa un valor que se asigna cuando no hay valores definidos para una variable. Indica la falta de asignación de valor.
    Operadores
    Permiten realizar diferentes tipos de operaciones, como operaciones matemáticas (+, -, *, /).
    Variables
    Espacios de almacenamiento que contienen datos. Permiten dar nombre a valores y referenciarlos posteriormente en el código.
    Explora las variables de JavaScript
    Aritméticos
    Operadores que realizan operaciones matemáticas, como suma, resta, multiplicación y división.
    Constantes
    Variables cuyos valores no pueden cambiar una vez asignados. Se declaran con la palabra clave const.
    Comparación
    Operadores que comparan dos valores y devuelven un resultado booleano, como mayor que (>), menor que (<), igual a (===), etc.
    Convención de nombres
    Reglas que se deben seguir al nombrar variables para mantener un código claro y comprensible. Incluye el uso de camel case y evitar palabras reservadas.
    Data types
    Tipos de datos en JavaScript, como números, strings, booleanos, objetos, etc.
    Let
    Palabra clave para declarar variables en JavaScript. Se utiliza para variables cuyo valor puede cambiar.
    Incremento y decremento
    Operadores (++ y --) utilizados para aumentar o disminuir el valor de una variable numérica en 1.
    Lógicos
    Operadores que realizan operaciones lógicas, como AND (&&), OR (||), y NOT (!).
    Métodos
    Funciones asociadas a objetos. Por ejemplo, alert() y prompt() son métodos utilizados para interactuar con el usuario.
    Reservadas
    Palabras clave en JavaScript que tienen un significado específico y no deben usarse como nombres de variables o funciones.
    Prompt
    Método que muestra un cuadro de diálogo al usuario para que ingrese información.
    Undefined
    Valor asignado a una variable que aún no ha sido inicializada con un valor específico.
    Var
    Palabra clave utilizada anteriormente para declarar variables en JavaScript. Ahora se prefiere el uso de let y const.
    Comprende qué son los operadores aritméticos
    Aritméticos
    • Incremento (++): Operador que incrementa el valor de una variable en uno.
    • Decremento (--): Operador que disminuye el valor de una variable en uno.
    • Suma (+): Operador que suma dos valores.
    • Resta (-): Operador que resta el segundo valor del primero.
    • Multiplicación (*): Operador que multiplica dos valores.
    • División (/): Operador que divide el primer valor por el segundo.
    • Exponencial ( o ^):** Operador que eleva el primer valor al exponente del segundo.
    Asignación
    • = (igual): Operador de asignación que asigna el valor a la variable.
    • += (más igual): Operador de asignación que suma el valor a la variable existente.
    • -= (menos igual): Operador de asignación que resta el valor de la variable existente.
    • *= (por igual): Operador de asignación que multiplica el valor a la variable existente.
    • /= (entre igual): Operador de asignación que divide el valor de la variable existente.
    Comparación
    • == (igual a): Operador de igualdad que devuelve verdadero si los valores son iguales, independientemente del tipo de datos.
    • === (igualdad estricta): Operador de igualdad estricta que devuelve verdadero solo si los valores y los tipos de datos son iguales.
    • != (no igual a): Operador de desigualdad que devuelve verdadero si los valores son diferentes.
    • !== (no igual estricto): Operador de desigualdad estricta que devuelve verdadero si los valores o los tipos de datos son diferentes.
    Lógicos
    • && (y): Operador lógico "y" que devuelve verdadero si ambas condiciones son verdaderas.
    • || (o): Operador lógico "o" que devuelve verdadero si al menos una de las condiciones es verdadera.
    • ! (no): Operador lógico "no" que invierte el valor de la condición.
    Data Structures: Arrays
    Concatenación
    Operación de unir dos o más strings. Se utiliza el operador de suma (+) para concatenar strings en JavaScript.
    Array
    Una variable especial que puede contener más de un valor. Los arrays son útiles para almacenar listas de elementos, y se accede a sus elementos mediante su índice numérico.
    Length
    Una propiedad o método de los arrays (y strings) en JavaScript que devuelve el número de elementos en el array o el número de caracteres en un string.
    Const
    Una palabra clave utilizada para declarar constantes en JavaScript. Sin embargo, con los arrays, no impide la modificación de los elementos del array, solo la reasignación del array completo.
    Multidimensional Array
    Un array que contiene uno o más arrays. Es una forma de crear una estructura con múltiples niveles de datos
    Object
    En JavaScript, los objetos son variables que pueden contener muchos valores. Estos valores son propiedades y constan de una clave (o nombre) y un valor. Los objetos se utilizan para representar entidades del mundo real y sus características.
    Data Structures: Objetos
    Corchetes
    Símbolos utilizados en JavaScript para definir arrays y acceder a sus elementos. También se usan en la creación de bloques de código.
    Índice
    La posición numérica de un elemento dentro de un array o estructura de datos. En JavaScript, los índices comienzan desde 0.
    Estructuras de datos
    Una palabra clave utilizada para declarar constantes en JavaScript. Sin embargo, con los arrays, no impide la modificación de los elementos del array, solo la reasignación del array completo.
    Métodos
    Funciones asociadas a objetos en JavaScript. Pueden realizar acciones o cálculos específicos relacionados con el objeto.
    Propiedades
    Características que describen un objeto en JavaScript. Cada propiedad tiene un nombre y un valor asociado.

    Explora las funciones con JavaScript
    Console.log:
    Una función en JavaScript utilizada para imprimir mensajes en la consola del navegador, útil para depurar y verificar resultados durante el desarrollo.
    Function:
    Una porción de código reutilizable que puede ser llamada en cualquier momento desde un programa para realizar una tarea o funcionalidad específica.
    JavaScript:
    Lenguaje de programación que se utiliza para agregar interactividad y dinamismo a las páginas web.
    Notación camelCase:
    Convención de escritura en la que las palabras dentro de un identificador comienzan con mayúsculas, excepto la primera palabra que comienza con minúscula.
    Parámetros:
    Valores que se pueden pasar a una función para que realice una tarea específica. Los parámetros se definen entre paréntesis en la declaración de la función.
    Return:
    Palabra clave utilizada para especificar el valor que una función debe devolver después de su ejecución.
    Conoce cómo usar condicionales
    and:
    Operador lógico que requiere que ambas condiciones especificadas sean verdaderas para que se cumpla la condición completa.
    break:
    Palabra reservada utilizada en el contexto de un switch para salir del bloque de código después de que se ha ejecutado un caso.
    else:
    Palabra reservada que se utiliza en una estructura condicional para especificar un bloque de código que se ejecutará si la condición no se cumple.
    if:
    Palabra reservada que inicia una estructura condicional en JavaScript, donde un bloque de código se ejecuta si la condición especificada es verdadera.
    switch:
    Palabra reservada que inicia una estructura condicional más compacta que puede reemplazar múltiples declaraciones if-else if.
    true:
    Valor lógico que indica que una condición es verdadera.
    false:
    Valor lógico que indica que una condición es falsa.
    ==:
    Operador de igualdad que compara si dos valores son iguales, sin tener en cuenta el tipo de dato.
